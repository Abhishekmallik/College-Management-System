College Management System

This is a project aiming to implement a utilitarian college management system using concepts of DataBase Management Systems. It uses SQLite as the database management system.

The various schemas designed and used are :

  • Departments :
     It contains attributes - deptName, Building & Budget, necessary to define a particular department.
  • Instructors :
     It contains attributes - id, name, salary & department_name, thereby giving details on a particular instructor. Here the department_name is a foreign key on the instructors table, from the department table, which enables us to query for the 'inst_dept' relationship set.
  • Students :
     It contains attributes - id, name, total_credits, instructor_name & department_name, enlisting the particulars of every student in the college. The Instructor ID is a foreign key on the students table which enables us to query for the 'advisor' relationship set. Also, The department name is a foreign key on the students table which enables us to query for the 'student_department' relationship set
  • Sections :
     It contains attributes - id, semester & year, thus uniquely defining a section. The "teaches" table is a relationship set between an instructor and section, and can be used to query which teacher teaches which section.

Various Relationship sets are also used in this implementaion, namely, inst_dept, advisor, student_department & lastly teaches. Out of all these relationship sets we only create 'teaches' schema explicitly as every other relationship set has a many-to-one relationship and the entity set with 'many' mapping also has 'full participation' so the relationship set is merged with that schema.
